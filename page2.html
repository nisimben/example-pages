Scrollable Content

This page lets you practice fixed and sticky positioning. The navigation bar at the top uses position: fixed and stays at the top as you scroll. The footer uses the same technique to remain at the bottom of the viewport.

The side note uses position: sticky. At first, it behaves like a normal block element, participating in the flow. When you scroll past it, it becomes fixed relative to the viewport and stays visible until you scroll past the end of its container.

Experiment by adjusting the top property of the sticky note. This property determines the point at which the note sticks. Try values like 0, 50px, or 100px to see how it changes the sticking behaviour.

Notice that the main content area has extra padding on the top and bottom. This ensures that the content doesn’t get hidden behind the fixed header and footer. Without this padding, part of your content would be obscured when the page loads.

Add more paragraphs or even images to make the page longer. The more content you have, the easier it is to see how the fixed and sticky elements behave while scrolling.

When you’re comfortable with fixed and sticky elements, move on to Page 3 where you’ll explore layering and z-index by creating overlapping elements.

Keep practicing and experimenting. Understanding positioning will help you build more complex layouts without relying on advanced layout systems like Flexbox or CSS Grid.

You can also try making your own sticky elements within this page. For example, try adding another sidebar that sticks after a different scroll threshold, or create a sticky header inside the article.

Remember: fixed elements are removed from the normal document flow and stay glued to the viewport. Sticky elements start as normal, then switch to fixed positioning after you scroll past them.
